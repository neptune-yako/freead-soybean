# SoybeanAdmin 环境切换说明

本项目基于 Vite 7 构建，支持通过不同的 `mode` 快速在本地开发、测试环境和生产环境之间切换。

## 1. 环境准备 (Node & PNPM)

为了保证项目正常运行，请确保你的本地环境满足以下最低要求（根据 `package.json` 指令）：

- **Node.js**: `v20.19.0` 或更高版本。
- **PNPM**: `v10.5.0` 或更高版本。

> [!IMPORTANT]
> 本项目**不建议**使用 `npm` 或 `yarn` 安装依赖，可能会导致依赖解析错误。请统一使用 `pnpm`。

---

## 2. 常用操作命令

### A. 依赖安装与启动

| 操作任务             | 命令                             | 说明                                 |
| :------------------- | :------------------------------- | :----------------------------------- |
| **安装所有依赖**     | `pnpm install`                   | 首次运行或换电脑后必做               |
| **启动开发服务器**   | `pnpm dev`                       | 启动开发模式（默认使用 `.env.test`） |
| **启动线上环境预览** | `pnpm dev:prod`                  | 使用生产配置启动开发模式             |
| **依赖重新安装**     | `pnpm cleanup` && `pnpm install` | 清除缓存并重新安装依赖               |

### B. 项目编译与构建

| 操作任务           | 命令              | 加载的环境文件 | 适用场景             |
| :----------------- | :---------------- | :------------- | :------------------- |
| **构建线上生产包** | `pnpm build`      | `.env.prod`    | 准备发布到正式服务器 |
| **构建测试环境包** | `pnpm build:test` | `.env.test`    | 准备发布到测试服务器 |

---

## 3. 环境配置文件说明

在项目根目录下，你会看到以下几个环境文件：

- **`.env`**: **基础配置（公共）**。无论运行哪个环境，都会首先加载此文件。建议存放应用标题、图标前缀、路由模式等通用设置。
- **`.env.test`**: **测试环境配置**。当执行 `pnpm dev` 时，默认加载此环境。
- **`.env.prod`**: **生产环境配置**。执行 `pnpm build` 或 `pnpm dev:prod` 时加载。

> [!TIP]
> 如果你需要添加不提交到 Git 的本地私有配置，可以创建 `.env.test.local` 或 `.env.prod.local` 文件，Vite 会自动优先加载它们。

## 2. 常用操作命令

根据 `package.json` 的定义，你可以使用以下命令进行环境切换：

| 操作任务             | 执行命令          | 加载的环境文件       | 适用场景                       |
| :------------------- | :---------------- | :------------------- | :----------------------------- |
| **启动开发模式**     | `pnpm dev`        | `.env` + `.env.test` | 日常本地功能开发与调试         |
| **模拟生产环境运行** | `pnpm dev:prod`   | `.env` + `.env.prod` | 在开发模式下测试生产接口逻辑   |
| **构建线上生产包**   | `pnpm build`      | `.env` + `.env.prod` | 准备发布到正式服务器的静态资源 |
| **构建测试环境包**   | `pnpm build:test` | `.env` + `.env.test` | 准备发布到测试/预发布服务器    |

## 3. 如何链接到你自己的后端接口

当你准备接入真实的后端服务时，请检查以下两个关键配置：

### A. 修改后端基础路径 (Base URL)

打开对应的环境文件（如 `.env.test`），修改 `VITE_SERVICE_BASE_URL`：

```bash
# .env.test
VITE_SERVICE_BASE_URL=http://192.168.1.100:8080/api/v1/admin
```

### B. 修改成功状态码 (Success Code)

在根目录的 **`.env`** 文件中，确认 `VITE_SERVICE_SUCCESS_CODE`。

- **注意**：本项目默认可能是 `0000`。如果你的后端参照 `AI_INSTRUCTIONS_FRONTEND.md` 返回 `200`，请务必将其修改为 `200`：

```bash
# .env
VITE_SERVICE_SUCCESS_CODE=200
```

---

## 4. 常见问题排查

- **环境变量未生效**：修改 `.env` 系列文件后，通常需要**重启开发服务器**（Ctrl + C 然后重新 `pnpm dev`）才能生效。
- **跨域问题**：开发环境下，项目已默认开启 `VITE_HTTP_PROXY=Y`。如果遇到跨域，请确认 `vite.config.ts` 中的 proxy 配置是否覆盖了你的接口路径。
